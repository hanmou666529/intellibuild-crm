# 数据大屏设计方案

## 背景
现有数据看板（9 图表 + 4 统计卡）升级为全屏大屏风格，用于大屏幕/电视展示。

## 需求
- 深色简洁商务风（非经典蓝光科技风）
- 1920×1080 满屏展示，无滚动
- 自动刷新（30 秒轮询）
- 点击图表下钻查看明细
- 单页，无需多页切换

### 说明
- 第一行：统计卡片（4 个）+ 时钟区域
- 第二行：3 环形图 + 柱状图（等高）
- 第三行：折线 + 横条 + 漏斗（等高）
- 第四行：跟进趋势通栏折线图

## 技术改动

### 前端
1. 新建 `ruoyi-ui/src/views/crm/dashboard/bigscreen.vue` — 大屏主组件
2. 路由 `src/router/modules/crm.js` — 添加 `/crm/dashboard/bigscreen` 路由
3. `bigscreen.vue` 内部：
   - `v-loading` 骨架屏
   - 自动刷新：`mounted` 中启动 `setInterval(30000)`，`beforeDestroy` 清除
   - ECharts 使用内置 `dark` 主题（`echarts.init(dom, 'dark')`）
   - 点击下钻：图表 `click` 事件 → 调用 drill-down API → Dialog 显示明细
   - 全屏样式覆盖 `body/html` 背景为 `#0f1117`

### 后端
4. 新增 `GET /crm/dashboard/drillDown` 接口（可选，可直接复用现有数据或无数据时提示）
   - `bizType`: `customer_source` | `order_status` | `contract_status` | `product_detail`
   - `key`: 具体的分类值
   - 返回明细列表

### 不修改
- 现有 `dashboard/index.vue` 保持不变
- 现有 API `/crm/dashboard/stats` 保持不变
- 现有后端 Service/Mapper 保持不变

## 实现顺序
1. 新建 `bigscreen.vue` + 路由
2. 深色主题样式 + 布局
3. 8 个 ECharts 图表移植（dark 主题）
4. 统计卡片（发光数字效果）
5. 自动刷新逻辑
6. 点击下钻 API + Dialog
7. 时钟 + 状态底栏
